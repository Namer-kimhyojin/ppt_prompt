# PPT 스크립트 우선 기획 스킬

스크립트가 먼저다. 슬라이드는 그 말을 돕는 도구다.

**이 스킬의 작업 순서:**

```
STEP 1. 핵심 메시지 확정        — 이 슬라이드에서 청중에게 남길 단 하나의 문장
STEP 2. 스크립트 초안 작성      — Hook → Body(SCR) → Bridge 구조로 말을 먼저 완성
STEP 3. 큐 목록 추출            — 스크립트에서 [화면 → X] 큐를 모두 뽑는다
STEP 4. 큐 → 레이아웃 요구사항  — 각 큐가 "슬라이드에 무엇이 어디 있어야 하는가"를 결정한다
STEP 5. AI 제작 명세 생성       — 큐 요구사항을 조합해 레이아웃 명세를 작성한다
```

> **역할 분담 원칙**
> - **이 스킬**: 스크립트·논리 흐름·정보 위계를 결정한다
> - **프롬프트덱**: 색상 팔레트·배경 명도·강조색을 결정한다
> - 레이아웃 명세에는 색상값·색상명을 쓰지 않는다. 위계는 크기·굵기·여백으로 표현한다.

---

## 1. 작업 시작 전 확인

컨텍스트에 이미 있는 정보는 재질문하지 않는다. 없는 것만 확인한다.

| 항목 | 기본값 |
|------|--------|
| 소스 문서 | 첨부 파일 또는 입력 텍스트 |
| 청중·목적 | 정부 심사위원회 |
| 발표 시간 | 슬라이드당 1분 기준 |
| 밀도 기본값 | L3 표준 (아래 참고) |
| 디자인 | 색상·폰트는 프롬프트덱 팔레트 따름 |

---

## 2. 발표 전략 수립

### 2-1. 청중 분석

| 청중 유형 | 핵심 관심사 | 스크립트 어조 |
|---------|-----------|-----------|
| 정부 심사위원 | 타당성·파급효과·실현가능성 | 수치 근거 + 리스크 인식 + 실행 의지 |
| 장관·고위직 | 핵심 결론·정책 임팩트 | 결론부터 — 3가지 이유 — 요청 |
| 업계·전문가 | 기술적 차별성·시장 논리 | 데이터 정밀도 + 업계 맥락 |
| 일반 시민·언론 | 생활과의 연결·직접 이익 | 쉬운 말 + 구체적 숫자 |

### 2-2. 논리 흐름 선택

```
[설득형]  Why → What → How → So What   ← 심사·승인
[보고형]  현황 → 성과 → 과제 → 계획     ← 업무보고
[제안형]  문제 → 기회 → 해법 → 요청     ← 정책 제안
```

### 2-3. 슬라이드 배분표

작업 전 전체 목록을 제시하고, 확인 없이 바로 작성을 시작한다.

```markdown
| No | 제목 | 파트 | 논리 위치 | 밀도 | 핵심 메시지 한 줄 |
|----|------|------|---------|------|----------------|
| 01 | 표지 | 도입 | — | L1 | {발표 제목 + 부제} |
...
```

## 3. 밀도 레벨

슬라이드마다 정보량을 의도적으로 조절한다. 밀도는 스크립트 분량과 직결된다.

| 레벨 | 명칭 | 스크립트 발화 수 | 화면 정보량 | 적합한 슬라이드 |
|------|------|--------------|-----------|--------------|
| **L1** | **임팩트** | 1~2문장 + 침묵 | 키워드 1~2개 | 표지·파트 전환·클로징 |
| **L2** | **집중** | 3~4문장 | 핵심 메시지 + 근거 2~3개 | 핵심 주장·Why Now |
| **L3** | **표준** | 4~5문장 | 주제 + 보조 항목 3~5개 | 전략·구조·현황 |
| **L4** | **정보** | 3~4문장 (핵심만) | 데이터·표 6개 이상 | 예산·비교·통계 |

**레벨 배분 가이드**
```
L1  ────  전체의 15~20%
L2  ────  전체의 25~30%
L3  ────  전체의 40~50%
L4  ────  전체의 10~15%
```

## 4. 슬라이드 1장 기획안 — 출력 형식

**STEP 1~5를 순서대로 실행한다. 스크립트를 완성하기 전에 레이아웃을 결정하지 않는다.**

```markdown
## 슬라이드 {NN}. {슬라이드 제목}

**제목:** {발표 화면에 표시될 타이틀}
**파트:** {part N. 파트명}
**페이지:** {NN} / {전체}
**논리 위치:** {Why / What / How / So What}
**밀도 레벨:** {L1~L4}

---

### STEP 1. 핵심 메시지
> **{청중이 이 슬라이드에서 단 하나만 기억해야 한다면 이 문장}**

---

### STEP 2. 스크립트

스크립트는 **큐 시트 형식**으로 작성한다.
[화면 → X] 큐는 "X가 슬라이드에 있어야 한다"는 레이아웃 요구사항이 된다.

연출 큐 기호:
- `[슬라이드 전환]` — 다음 슬라이드로 넘긴다
- `[화면 → {요소}]` — 해당 요소를 포인팅한다 ← **이 큐가 레이아웃을 결정한다**
- `[침묵 N초]` — 청중이 받아들이는 시간
- `[청중 시선]` — 화면에서 눈을 떼고 청중을 바라본다
- `[속도: 느리게/빠르게]` — 발화 속도 조절

```
🪝 Hook
[청중 시선]
"{결론·수치·대비 — 한 문장. 화면이 아닌 청중을 향해.}"
[침묵 1~2초]

📖 Body
[화면 → {요소 A}]
"{덩어리 ①: ~지만 / ~가운데 / ~이지만 으로 맥락을 한 문장에.}"

[화면 → {요소 B}]
"{덩어리 ②: ~는 순간 / ~되면 으로 조건-결과를 한 문장에.}"
[침묵 1초]

"{덩어리 ③: 짧고 단호하게.}"

🌉 Bridge
[청중 시선]
[속도: 느리게]
"{①닫기: 이 슬라이드 결론 한 문장.}"
"{②열기: 다음 슬라이드로 이어지는 한 문장.}"
[슬라이드 전환]
```

---

### STEP 3. 큐 목록 추출

스크립트에서 `[화면 → X]` 큐를 모두 추출한다.

| 순서 | 큐 | 발화 시점 | 화면에 필요한 것 |
|------|---|---------|--------------|
| ① | `[화면 → {요소 A}]` | "{직전 발화 문장 앞부분}" | {어떤 정보가, 어느 크기로, 어디에} |
| ② | `[화면 → {요소 B}]` | "{직전 발화 문장 앞부분}" | {어떤 정보가, 어느 크기로, 어디에} |

---

### STEP 4. 레이아웃 요구사항

큐 목록에서 도출한 "슬라이드에 있어야 할 것들"을 정리한다.

**필수 요소 목록:**
- ★ 시선 시작점: {요소 A} — {이유: 스크립트 Hook과 연동, 가장 먼저 눈에 들어와야}
- ★ 두 번째 강조: {요소 B} — {이유: Body 덩어리 ②와 연동}
- 보조 요소: {나머지}

**레이아웃 유형 선택:**
> {위 요소 목록을 담기 가장 적합한 레이아웃 유형} — 선택 이유 한 줄
> (레이아웃 유형 라이브러리는 섹션 5 참고)

---

### STEP 5. AI 제작 명세

> **색상 지정 없음** — 배경 명도·팔레트·강조색은 프롬프트덱에서 결정한다.
> 위계는 크기·굵기·여백으로만 표현한다.

```
[SLIDE LAYOUT]
분할 방식: {좌3:우7 / 상하 2분할 / 3×2 그리드 / 전면 풀스크린 등}
배경 처리: {배경과 콘텐츠의 대비 강도 — 최고 대비 / 동등한 무게 / 낮은 무게감 — 색상은 팔레트 따름}

[SECTION 1 — {영역명}]
위치: {상단 전체 / 좌측 N% / 우측 N% 등}
높이: 화면의 {N}%
콘텐츠 유형: {빅타이포 / 도넛차트 / Bento카드 / 타임라인 / 흐름도 등}
내용:
  - {요소 A}: {텍스트·수치}  ★시선 시작점 — 가장 크게·강하게
  - {요소 B}: {텍스트·수치}  ★두 번째 강조
  - {나머지}: {텍스트}       (보조 정보, 조용하게)
정렬: {좌 / 중앙 / 우}
스크립트 연동: {이 섹션의 어느 큐와 연결되는가}

[SECTION 2 — {영역명}]
...

[강조 원칙]
- 시선 시작점(★): {요소 A} — 슬라이드당 1개
- 두 번째 강조(★): {요소 B} — 1~2개
- 나머지: 보조 정보로 조용하게
- ※ 색상 지정은 프롬프트덱에서 수행. 크기·굵기·여백으로 위계를 표현한다.

## 5. 레이아웃 유형 라이브러리

STEP 4에서 레이아웃 유형을 선택할 때 이 라이브러리를 참조한다.

> **색상 분리 원칙:** 레이아웃 유형명과 구조 설명은 형태·위계만 기술한다.
> 배경 명도·팔레트·강조색은 프롬프트덱의 역할이다.

### 임팩트·선언형 (L1·L2)

| 유형 | 구조 | 적합한 슬라이드 | 대표 큐 패턴 |
|------|------|--------------|-----------|
| **풀임팩트** | 전면 고대비 배경 + 대형 타이포 + 넓은 여백 (배경은 콘텐츠 대비 최고 무게감, 명도·색상은 팔레트 따름) | 표지·파트 전환·클로징 | `[화면 → 제목 타이포]` 1개 |
| **빅타이포 임팩트** | 핵심 수치 1개가 화면 40~50% 차지 + 보조 Bento 카드 | Why Now·충격 수치 | Hook 직후 `[화면 → 수치]` |
| **풀블리드 인용** | 배경 이미지 전면 + 반투명 오버레이 + 한 줄 인용문 | 비전 선언·클로징 | `[화면 → 인용문]` 1개 |
| **카운트다운 카드** | 숫자 3~4개 대형 나열 + 레이블 | 3대 이유·4대 전략 | `[화면 → ①]` `[화면 → ②]` `[화면 → ③]` |

### 대비·비교형 (L2·L3)

| 유형 | 구조 | 적합한 슬라이드 | 대표 큐 패턴 |
|------|------|--------------|-----------|
| **대비 분할** | 좌우 두 영역 (시각 무게감 대비로 문제↔해법 구분, 명도 차이는 팔레트 따름) | 현황 vs 목표 | `[화면 → 좌측]` → `[화면 → 우측]` |
| **Before·After 스플릿** | 상하 또는 좌우로 시간 전후 분리 | 정책 효과·전후 변화 | `[화면 → Before]` → `[화면 → After]` |
| **경쟁 매트릭스** | 2×2 또는 3×3 그리드 비교 | 경쟁사·입지·기술 비교 | `[화면 → 우리 위치]` |
| **선택지 카드** | 옵션 2~3개 병렬 + 추천 옵션 위계 강조 | 전략 선택·시나리오 | `[화면 → 추천 옵션]` |

### 구조·흐름형 (L3)

| 유형 | 구조 | 적합한 슬라이드 | 대표 큐 패턴 |
|------|------|--------------|-----------|
| **서사 흐름** | 수평 5단계 플로우 | 프로세스·밸류체인 | `[화면 → N단계]` 순서대로 |
| **허브-스포크** | 중심 노드 + 4~6개 방사형 | 거버넌스·협력체계 | `[화면 → 중심]` → `[화면 → 연결 요소]` |
| **타임라인 드라마** | 수평 시간 축 + 노드별 카드 | 로드맵·추진일정 | `[화면 → N년 노드]` |
| **3단 논거** | 상단 주장 + 기둥 3개 + 하단 결론 | 차별성·3대 이유 | `[화면 → ①]` `[화면 → ②]` `[화면 → ③]` |
| **피라미드·계층** | 상위→하위 계층 구조 | 조직도·정책 체계 | `[화면 → 최상위]` → `[화면 → 하위]` |
| **순환 사이클** | 원형 순환 고리 + 단계 레이블 | 생태계 선순환 | `[화면 → 진입점]` |
| **스윔레인** | 행별 주체 + 열별 단계 | 협업 프로세스 | `[화면 → 주체 행]` |

### 공간·지역형 (L3)

| 유형 | 구조 | 적합한 슬라이드 | 대표 큐 패턴 |
|------|------|--------------|-----------|
| **지도 + 카드** | 상단 지도 + 하단 팩트 카드 | 입지·권역 설명 | `[화면 → 위치]` → `[화면 → 팩트]` |
| **클러스터 맵** | 지도 위 거점·연결선 오버레이 | 산업 클러스터·물류망 | `[화면 → 거점]` |

### 데이터·분석형 (L3·L4)

| 유형 | 구조 | 적합한 슬라이드 | 대표 큐 패턴 |
|------|------|--------------|-----------|
| **도넛 + Bento** | 좌: 도넛차트 + 우: 항목 카드 | 예산 배분·구성 요약 | `[화면 → 도넛]` → `[화면 → 카드]` |
| **데이터 해부** | 중앙 70% 표·차트 + 우측 30% 코멘트 | 현황표·성과 비교 | `[화면 → 핵심 셀]` |
| **트렌드 라인** | 꺾은선·영역 차트 + 이벤트 마커 | 시장 성장·추이 | `[화면 → 변곡점]` |
| **히트맵 그리드** | 행·열 교차점을 채움 강도로 표현 (강도 방식은 팔레트 따름) | 지역별·품목별 현황 | `[화면 → 고강도 셀]` |
| **스코어카드** | KPI 항목별 현재값·목표값·달성률 | 성과 현황·목표 대비 | `[화면 → 달성률]` |
| **모자이크 타일** | 크기 다른 타일로 비중·중요도 표현 | 시장 점유율·예산 비중 | `[화면 → 최대 타일]` |

## 6. 스크립트 작성 원칙

### 6-0. 이론적 기반

| 프레임워크 | 출처 | 이 스킬에서 담당하는 역할 |
|----------|------|----------------------|
| **SCR** (Situation–Complication–Resolution) | McKinsey | 스크립트 흐름 — 현실→문제→해법 |
| **Pyramid Principle** (결론 먼저) | Barbara Minto | Hook — 핵심 메시지를 먼저 던진다 |
| **Oral Style** (구어체) | 언어학 연구 | 문장 단위 — 들리는 말로 쓴다 |

### 6-1. 어조 목표: "공식적이되 자연스러운 구어체"

| 특성 | ❌ 나쁜 예 | ✅ 좋은 예 |
|------|---------|---------|
| 구체적 주어 | "추진되어 왔으며 향후에도 지속될 예정입니다." | "포항시가 추진해 왔고, 앞으로도 이어갑니다." |
| 덩어리 문장 | "만듭니다. / 옵니다. / 멈춥니다." | "만들지만, 소재는 중국에서 들어오고 있습니다." |
| 결론 먼저 | "LFP가 성장하고 있어 대응이 필요합니다." | "LFP 시장은 3년 만에 두 배로 커졌습니다." |
| 쉬운 어휘 | "구조적 취약성" | "언제든 흔들릴 수 있는 구조" |

### 6-2. 피할 표현

| ❌ 피할 표현 | ✅ 대신 이렇게 |
|------------|--------------|
| "말씀드리자면", "사실은" | 바로 본론으로 |
| "~이라고 할 수 있겠습니다" | "~입니다" |
| "그리고", "또한", "아울러" | 덩어리로 묶거나 마침표로 끊기 |
| "보시다시피 차트에서..." | 수치의 의미를 말한다 |
| "본 단지는", "해당 사업은" | "저희는", "포항은" |
| "이상으로 발표를 마치겠습니다" | 행동 요청으로 끝낸다 |
| "가장 시급합니다", "유일한 답" | 사실을 말하면 청중이 판단한다 |
| "추진되어 왔으며 향후에도" | 능동형으로 바꾼다 |

### 6-3. SCR 덩어리 문장 작성법

```
❌ 선언 나열
"배터리는 국내에서 만듭니다."
"핵심 소재는 중국에서 옵니다."
"공급이 막히면 생산이 멈춥니다."

✅ SCR 덩어리
"배터리는 국내에서 만들지만,
 핵심 소재의 대부분은 중국에서 들어오고 있습니다."   ← S: 현실 (대비를 한 문장에)

"이 공급망이 막히는 순간,
 이차전지 생태계 전체가 멈춰버립니다."              ← C: 문제 (조건-결과를 한 문장에)

"지금 우리에게는 소재를 직접 만들 수 있는 기반이 필요합니다."  ← R: 해법
```

| 관계 | 연결 방식 | 예시 |
|------|---------|------|
| 대비 S | `~지만, ~` | "갖췄지만, 원료 자립도는 5%입니다." |
| 조건-결과 C | `~는 순간, ~` | "흔들리는 순간, 라인 전체가 멈춥니다." |
| 배경-핵심 S | `~가운데, ~` | "커지는 가운데, 의존도는 95%입니다." |
| 나열-결론 R | `세 가지 모두` | "입지·인프라·인력이 포항에 있습니다." |

### 6-4. Hook 유형

| 유형 | 예시 |
|------|------|
| 충격 수치형 | "95%. 핵심 배터리 원료의 중국 의존도입니다." |
| 결론 선언형 | "소재 자립 없이는 배터리 강국도 없습니다." |
| 대비형 | "배터리는 만들 수 있습니다. 원료는 만들지 못합니다." |
| 질문형 | "그 원료를, 지금 어디서 가져오고 있을까요?" |
| 공감형 | "공급망 위기는 뉴스 속 이야기가 아닙니다." |

### 6-5. Bridge 2단 구조

Bridge는 반드시 ①현재 슬라이드 닫기 + ②다음 슬라이드 열기로 구성한다.

| 상황 | ①닫기 | ②열기 |
|------|------|------|
| 문제→해법 | "소재 자립. 그것이 포항이 해야 할 일입니다." | "포항은 그 조건을 이미 갖추고 있습니다." |
| 전략→실행 | "전략의 방향은 명확합니다." | "이제 실현할 구체적인 계획을 보여드리겠습니다." |
| 근거→결론 | "세 조건이 모두 포항에 있습니다." | "남은 질문은 하나입니다 — 언제 시작하느냐." |

### 6-6. 발표 리듬

```
[고조]   위기·긴급성 슬라이드  — 짧은 문장, 빠른 호흡
[안정]   해법·구조 슬라이드    — 차분하고 명확한 설명
[정점]   핵심 임팩트 슬라이드  — 전환 후 1~2초 침묵, 한 문장
[마무리] 클로징 슬라이드      — 가장 느린 속도, 행동 요청
```

## 8. 품질 체크리스트

**순서**
- [ ] 스크립트를 완성하기 전에 레이아웃을 결정하지 않았는가
- [ ] STEP 3 큐 목록이 STEP 5 명세의 섹션과 1:1로 대응되는가

**스크립트**
- [ ] Hook이 청중을 향한 형태인가 (화면 낭독 아님)
- [ ] Body가 2~3개 SCR 덩어리인가 (잘게 끊긴 나열 아님)
- [ ] Bridge가 ①닫기 + ②열기 2단 구조인가
- [ ] 군더더기 표현·수사적 선언이 없는가 (6-2 참고)
- [ ] 피동형·공문서체가 능동형·구어체로 바뀌었는가
- [ ] 클로징이 요약이 아닌 행동 요청으로 끝나는가

**큐 → 레이아웃 연동**
- [ ] `[화면 → X]` 큐마다 STEP 3 큐 목록에 항목이 있는가
- [ ] STEP 3의 "화면에 필요한 것"이 STEP 5 명세의 콘텐츠에 반영됐는가
- [ ] 시선 시작점(★)이 스크립트의 Hook/첫 번째 큐와 일치하는가
- [ ] AI 명세에 색상값·색상명이 없는가 (색상은 프롬프트덱 역할)

**밀도·리듬**
- [ ] 슬라이드 목록에 밀도 레벨이 배정됐는가
- [ ] 같은 레벨이 3장 이상 연속되지 않는가
- [ ] 전체 발표가 고조→안정→정점→마무리 리듬을 따르는가
- [ ] MD 파일로 출력됐는가
