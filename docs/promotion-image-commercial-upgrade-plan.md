# 홍보용 이미지 프롬프트 상업화 고도화 계획

## 1. 문서 목적

이 문서는 현재 구현된 `홍보용 이미지` 탭을 기준으로,  
프롬프트 품질을 상업적 수준으로 끌어올리고 사용자 혼란은 낮추는 방향의
세부 구현 계획을 정리한다.

단순히 기능을 많이 추가하는 것이 아니라 아래 3가지를 동시에 달성하는 것을 목표로 한다.

- 더 높은 광고/브랜드 캠페인 수준의 이미지 결과물 유도
- 더 창의적이고 덜 템플릿스러운 디자인 생성
- 기능 과밀로 인한 사용자 혼란 최소화

---

## 2. 현재 구현 상태 요약

현재 홍보용 이미지 탭은 이미 다음 기능을 갖고 있다.

- 단일 `홍보 이미지` 흐름으로 통합된 입력 구조
- 템플릿 기반 콘텐츠 유형 선택
- 비율/직접 크기 입력
- 헤드라인/서브카피/본문/CTA 입력
- 브랜드 톤/비주얼 스타일/품질 보정/배경 지시 프리셋
- 색상 입력, 컬러피커, 색상 프리셋 저장/적용/삭제
- 메인 비주얼 포인트, 정보 배치 방식, 첫 줄 훅, 해시태그, 비주얼 중심 포인트, 배치 메모
- 빈 항목 제거, 중복 제거, 충돌 자동 정리
- 한글/영문/병기 출력
- 검토용/최적화용 프롬프트 출력
- 변형 방향 선택
- 프롬프트 직접 편집/원복
- 샘플 입력, 프리셋 랜덤, 저장/불러오기
- 스모크 테스트 자동 검증

즉, 현재 단계는 “기본 빌더 부재”가 아니라  
“이미 기능은 충분히 많고, 이제 품질과 사용성의 정교화가 필요한 단계”다.

## 3. 현재 구조의 강점과 한계

### 3.1 강점

- 프롬프트를 `직접 노출 텍스트`와 `설계 지시`로 분리한 구조가 좋다.
- 입력값 정리, 중복 제거, 금지 규칙 유지 같은 실무형 보조 기능이 있다.
- 색상, 배경, 배치, 톤을 별도로 다룰 수 있어 조정 폭이 넓다.
- 영문 최적화 프롬프트까지 지원해 모델 친화성이 높다.

### 3.2 한계

- 기본 프롬프트가 아직 `고해상도/가독성` 중심이며 `광고 아트디렉션` 강도는 부족하다.
- 창의성 관련 입력이 `비주얼 스타일`, `변형 방향`, `메인 비주얼 포인트`에 분산되어 있다.
- “브랜드급 결과”를 원하는 사용자가 어떤 항목부터 건드려야 할지 직관적이지 않다.
- 고급 기능을 전면에 다 올리면 초심자에게 복잡하게 느껴질 수 있다.
- 구 기획의 흔적이 일부 코드/문서에 남아 있어 구조 정리 필요성이 있다.

## 4. 핵심 문제 정의

현재 가장 중요한 문제는 아래 2개다.

### 4.1 결과 품질 문제

사용자가 입력을 많이 하지 않아도  
기본 프롬프트가 `상업 광고 수준의 고급스러운 결과`를 더 강하게 유도해야 한다.

현재 기본 프롬프트는 다음에는 강하다.

- 해상도
- 텍스트 선명도
- 대비
- 저해상도 방지

하지만 아래 요소는 상대적으로 약하다.

- 광고 아트디렉션
- 브랜드급 마감
- 시각적 개성
- 강한 포컬 포인트
- 고급 편집 구성
- 창의적 레이아웃

### 4.2 사용성 문제

상업적 고도화를 위해 기능을 모두 추가하면  
사용자는 아래와 같은 혼란을 겪을 수 있다.

- 브랜드 톤 vs 비주얼 스타일 vs 변형 방향 차이를 모름
- 창의성 관련 옵션이 서로 겹쳐 보임
- 어떤 항목이 “필수”이고 어떤 항목이 “고급”인지 구분이 어려움
- 결과가 안 좋을 때 무엇부터 수정해야 하는지 판단하기 어려움

따라서 기능 추가보다 먼저 `노출 구조 설계`가 중요하다.

## 5. 제품 방향 원칙

이번 고도화는 아래 원칙을 따른다.

### 5.1 기본은 단순하게

초심자는 기본 입력 몇 개만으로도  
상업적으로 준수한 결과를 얻을 수 있어야 한다.

### 5.2 고급 기능은 숨기되 강하게

전문 사용자를 위한 기능은 충분히 제공하되  
기본 화면에 모두 펼쳐놓지 않는다.

### 5.3 “좋아 보이는 이유”를 시스템화

좋은 홍보 이미지는 보통 아래 요소에서 나온다.

- 강한 포컬 포인트
- 명확한 타이포 위계
- 제한된 색상 체계
- 레이아웃 긴장감
- 여백과 정돈감
- 브랜디드한 재질/광원/배경 설계

이 요소들을 프롬프트 구조와 UI에 반영해야 한다.

### 5.4 창의성은 “랜덤”이 아니라 “방향성”으로 제어

창의성은 무작위 스타일 나열이 아니라 아래 방식으로 제어한다.

- 조형 방향
- 구도 방향
- 아이디어 은유
- 브랜드 무드
- 광고 마감 수준

- 정보 배치 방식

### 6.2 상업 품질 레이어

기본 접힘, 필요 시 확장

기본 접힘

- 추천 수정 액션

## 7. 기본 프롬프트 구조 개선안

현재 기본 프롬프트는 품질 보정 쪽 비중이 크다.  
이를 아래 구조로 재설계한다.

### 7.1 새 기본 프롬프트 7단 구조

1. Role
2. Commercial Baseline
3. Output Target
4. Hard Constraints
5. Exact Text
6. Creative Direction
7. Color / Background / Production Finish

### 7.2 기본 자동 포함 문구 강화 방향

사용자 입력이 거의 없어도 아래 핵심 지시는 기본 포함되도록 한다.

- premium campaign-grade promotion image
- non-generic editorial advertising composition
- one dominant focal idea
- strong typography hierarchy
- generous whitespace
- high production polish
- sophisticated commercial finish
- avoid generic social thumbnail aesthetics

### 7.3 기본 포함 문구의 목표

기본값만으로도 아래 인상이 나와야 한다.

- “단순한 안내 이미지”보다 “브랜드 캠페인 컷”
- “정보 슬라이드”보다 “광고 시안”
- “예쁜 디자인”보다 “상업적으로 정리된 이미지”

## 8. 신규 기능 제안 상세

## 8.1 Commercial Baseline

### 목적

사용자 입력이 적어도 결과가 광고 수준으로 가도록 기본 바닥 품질을 끌어올린다.

### UI

- 위치: 상업 품질 레이어
- 형태: 토글 + 강도 선택
- 옵션:
  - Off
  - Standard
  - Premium
  - Luxury

### 프롬프트 반영

- Standard: 선명도, 정돈감, 광고 마감
- Premium: 브랜드 광고 수준의 완성도, 정교한 재질/광원
- Luxury: 고급 편집형, 절제된 구성, 하이엔드 캠페인 톤

### 구현 포인트

- `DEFAULT_STATE`에 `commercialBaseline`
- `FIELD_LABELS`, `FIELD_LABELS_EN` 추가
- `createPromptSections()`에 `commercialLines` 섹션 추가
- `renderOptimizedPrompt()`에서 intro 직후 우선 반영

---

## 8.2 창의성 강도

### 목적

사용자가 “안정적 결과”와 “실험적 결과” 사이를 쉽게 조절할 수 있게 한다.

### UI

- 위치: 창의성 레이어
- 형태: segmented control
- 옵션:
  - 안정형
  - 균형형
  - 실험형

### 프롬프트 반영

- 안정형: 읽기 쉬운 구성, 위험도 낮음
- 균형형: 창의성과 실용성 균형
- 실험형: 비대칭, 오버랩, bleed, 강한 구조 변형 허용

### 구현 포인트

- 기존 `variationMode`와 연동
- `VARIATION_SEEDS`를 `창의성 강도`에 맞게 재구성
- 단순 문구가 아니라 레이아웃/시선/여백 규칙까지 포함

---

## 8.3 Big Idea

### 목적

이미지 전체를 하나의 핵심 개념으로 묶는다.

### 예시

- 연결
- 확장
- 돌파
- 신뢰
- 참여
- 전환
- 희소성
- 속도
- 미래

### UI

- 위치: 창의성 레이어
- 형태: 입력 + 빠른 프리셋 버튼

### 프롬프트 반영

- 메인 비주얼
- 광원 방향
- 재질감
- 배경 구조
- 카피 톤

을 하나의 상징 방향으로 통일

### 구현 포인트

- 새 state: `bigIdea`
- 새 quick preset group
- lint에서 `bigIdea` 없이 스타일만 많은 경우 경고

---

## 8.4 비주얼 은유 프리셋

### 목적

창의성의 질을 높인다.  
스타일보다 “무엇을 상징적으로 보여줄지”가 결과 퀄리티에 더 중요하다.

### 예시 프리셋

- 유리 큐브
- 빛의 축
- 층위가 겹치는 구조
- 절단면/단면
- 데이터 흐름
- 프레임 돌출
- 구조물의 교차
- 확장되는 링
- 압축된 빛

### UI

- 위치: 창의성 레이어
- 형태: quick buttons + 자유 입력

### 구현 포인트

- 새 state: `visualMetaphor`
- `main visual focal point`와 충돌 시 lint 경고
- optimized prompt에서는 명사형보다 directive형으로 재서술

---

## 8.5 광고 마감 전용 설정

### 목적

“선명한 이미지”를 넘어서 “광고 시안 같은 마감”을 유도한다.

### 세부 항목

- 광원 톤
- 재질감
- 카메라 거리
- 렌더링 밀도
- 표면 질감
- 인쇄용 마감감

### UI

- 위치: 상업 품질 레이어
- 형태: 접힘 섹션

### 구현 포인트

- 새 state 묶음:
  - `lightingStyle`
  - `materialStyle`
  - `cameraDistance`
  - `renderFinish`
- 기본 프리셋 제공

---

## 8.6 Anti-Generic Lint

### 목적

결과가 흔해질 가능성을 사전에 경고한다.

### 검사 규칙 예시

- 포컬 포인트 없음
- 스타일 키워드만 많고 개념이 없음
- 색상 지시 과다
- 본문 포인트 과다
- CTA 약함
- 톤과 비주얼 스타일 충돌
- 상업 품질 지시가 너무 약함

### 구현 포인트

- `detectPromptLint()` 확장
- `genericityScore` 계산
- lint 패널에 “무난함 위험” 표시

---

## 8.7 3안 자동 변주 생성

### 목적

사용자가 한 입력으로 여러 수준의 창의안을 빠르게 비교할 수 있게 한다.

### 출력 유형

- 타이포 중심안
- 오브젝트 중심안
- 실험 레이아웃안

### UI

- 위치: 창의성 레이어 또는 실험 영역
- 형태: `3안 생성` 버튼

### 구현 포인트

- 기존 `VARIATION_SEEDS` 활용
- 미리보기 탭 또는 카드 3개 생성
- 복사 버튼은 각 안별 제공

## 12. 수용 기준

아래를 만족하면 이번 고도화가 유효하다고 본다.

- 기본 입력만으로도 프롬프트가 더 상업 광고 지향적으로 바뀐다.
- 사용자가 추가 기능 때문에 첫 화면에서 압도되지 않는다.
- 창의성 관련 항목이 서로 역할이 겹치지 않는다.
- 영문 최적화 프롬프트에서도 신규 기능이 자연스럽게 번역된다.
- lint가 “무난함 위험”과 “과밀 위험”을 실질적으로 잡아준다.
- 스모크 테스트가 신규 기능을 포함해 통과한다.

## 14. 결론

현재 홍보용 이미지 탭은 이미 기능이 많은 편이므로  
앞으로의 개선은 “더 많은 입력창 추가”보다  
`기본 프롬프트의 상업적 수준 강화`, `창의성의 방향성 제어`, `단계적 UI 노출`이 핵심이다.

따라서 이번 고도화는 아래 전략으로 진행한다.

- 기본 결과물의 수준을 올리는 기능부터 먼저 구현
- 창의성 기능은 방향 기반으로 묶어 제공
- 전문가용 기능은 접힘/실험실 구조로 분리
- lint와 추천으로 사용자의 판단 부담을 줄임

이 방향이 가장 결과 품질과 사용자 친화성을 함께 만족시킬 가능성이 높다.
