# 홍보 이미지 기획 탭 v2 구현계획

## 1. 목적

단순 홍보용 이미지 제작을 넘어 `홍보 이미지 기획`에 특화된 이미지 프롬프트 생성기 제작.

핵심 사용 목적은 아래 4가지를 우선한다.

- 사업공고
- 행사안내
- 수요/설문조사
- 정책/정보전달

핵심 이미지 형태는 아래 3가지를 우선한다.

- 포스터 (전단지 포함)
- SNS 이미지
  - 카카오채널 메시지
  - 일반 SNS 정방형/세로형 이미지
- 배너 (미니배너, 일반배너, 웹사이트 메인배너 포함)

## 2. 설계 원칙

- 기존 `홍보용 이미지` 탭은 변경하지 않는다.
- 새 탭은 기존 탭과 완전히 분리한다. `promotion.js`의 상태, DOM, 클래스, 생성 로직에 의존하지 않는다.
- 시각 일관성을 위한 공통 폼 스타일은 재사용할 수 있지만, 기획 로직과 ID/클래스 네임스페이스는 `planner` 기준으로 유지한다.
- 새 탭은 사용자가 한 문장으로 시작할 수 있어야 한다.
- `빅아이디어`와 `비주얼 은유`를 중심 기능으로 둔다.
- 질문 항목을 줄이고, 상충 가능성이 있는 항목은 통합한다.
- 선택지는 핵심 목적과 핵심 이미지 형태에 최적화한다.
- 최종 프롬프트는 이미지 생성 API로 바로 넘길 수 있는 섹션형 텍스트로 만든다.
- 다크모드/라이트모드를 모두 지원한다.

## 3. 추천 사용자 흐름


## 8. 핵심 목적 선택지

- 사업공고
- 행사안내
- 수요/설문조사
- 정책/정보전달
- 일반 홍보

목적별 기본값:

- 사업공고: 신뢰감, 정보 구조, 대상/혜택/마감 강조
- 행사안내: 일정, 장소, 신청 CTA, 현장감
- 수요/설문조사: 참여 이유, 응답 혜택, 간단한 절차, 신뢰감
- 정책/정보전달: 핵심 메시지, 단계형 정보, 가독성, 공신력
- 일반 홍보: 브랜드/서비스 인지도와 CTA 균형

## 10. 빅아이디어 / 비주얼 은유 정책

이 기능은 새 탭의 차별점으로 유지한다.

- 빅아이디어는 메시지의 상징 방향이다.
- 비주얼 은유는 실제 이미지로 표현될 장면/오브젝트다.
- 스타일과 분리해 상충을 줄인다.

예:

```text
Big idea: 연결과 성장
Visual metaphor: 빛의 길이 여러 정보 카드와 사람을 연결하는 장면
```

## 11. 상충 방지 룰

1차 룰:

- CTA 강조 선택 + CTA 비어 있음
- 일정/마감 강조 선택 + 일정 비어 있음
- 포스터 선택 + 정보 항목이 너무 적음
- SNS 선택 + 본문 정보가 너무 많음
- 배너 선택 + 정보 항목이 너무 많음
- 미니멀 스타일 + 정보 항목이 많음
- 어두운 배경 + 색상 직접 지정이 어두운 색 위주
- 빅아이디어/비주얼 은유가 모두 비어 있음 + 창의적 스타일 선택
