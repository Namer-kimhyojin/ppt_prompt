# 홍보이미지 기능 개발 계획

## 1. 목적

기존 `발표자료 프롬프트 설계` 기능과 별도로 `홍보이미지 제작` 작업군을 추가한다.  
대상 산출물은 다음 3가지로 시작한다.

- 포스터
- 카드뉴스
- SNS 홍보이미지

핵심 목표는 다음과 같다.

- 발표자료용 설정과 홍보이미지용 설정을 분리한다.
- 일반 사용자가 작업 목적에 맞는 입력만 보도록 UI를 단순화한다.
- 공통 자산은 재사용하고, 프롬프트 생성 규칙은 작업군별로 독립시킨다.
- 저장/불러오기와 자동 스모크 테스트까지 포함해 안정적으로 확장한다.

---

## 2. 추천 정보 구조

### 2.1 최상위 탭

- 발표자료
- 슬라이드별 생성기
- 홍보이미지

### 2.2 홍보이미지 내부 분류

- 포스터
- 카드뉴스
- SNS 홍보이미지

### 2.3 분리 원칙

발표자료와 홍보이미지는 공통으로 `브랜드 톤`, `색상`, `타이포`, `비주얼 스타일`, `금지 규칙`을 사용할 수 있다.  
하지만 아래 개념은 발표자료 전용으로 유지한다.

- 페이지 유형
- TOC
- 부록
- 슬라이드별 프롬프트 편집

홍보이미지는 별도 개념으로 다룬다.

- 채널 목적
- 카피 구조
- CTA
- 플랫폼 규격
- 컷 수
- 시리즈 흐름

---

## 3. MVP 범위

### 3.1 포함

- 최상위 `홍보이미지` 탭 추가
- 내부 서브 탭 `포스터 / 카드뉴스 / SNS 홍보이미지`
- 홍보이미지 전용 입력 폼
- 홍보이미지 전용 프롬프트 생성 미리보기
- 홍보이미지 전용 프리셋 3~5종
- 홍보이미지 설정 저장/불러오기
- 자동 스모크 테스트 1차 추가

### 3.2 제외

- 이미지 생성 API 직접 연동
- 카드뉴스 컷별 개별 편집 모달
- 템플릿 마켓/공유 기능
- 플랫폼별 고급 안전 필터

---

## 4. 사용자 흐름

### 4.1 포스터

1. `홍보이미지` 탭 진입
2. `포스터` 선택
3. 캠페인 목적, 타깃, 헤드라인, 서브카피, CTA 입력
4. 비율/톤/비주얼 스타일 선택
5. 생성 프롬프트 확인 및 복사

### 4.2 카드뉴스

1. `홍보이미지` 탭 진입
2. `카드뉴스` 선택
3. 컷 수 선택
4. 표지 카피, 컷별 요점, 마지막 CTA 입력
5. 시리즈 프롬프트 또는 컷별 프롬프트 확인

### 4.3 SNS 홍보이미지

1. `홍보이미지` 탭 진입
2. `SNS 홍보이미지` 선택
3. 플랫폼 선택
4. 핵심 문구, 보조 문구, CTA, 해시태그/주의사항 입력
5. 규격에 맞는 프롬프트 확인

## 5. 데이터 모델 초안

홍보이미지 전용 상태는 별도 객체로 관리한다.

```js
promotionState = {
  assetType: "poster" | "cardnews" | "sns",
  platform: "instagram-feed" | "instagram-story" | "facebook" | "blog" | "custom",
  ratio: "1:1" | "4:5" | "9:16" | "16:9" | "custom",
  goal: "",
  audience: "",
  headline: "",
  subheadline: "",
  bodyCopy: "",
  cta: "",
  tone: [],
  visualStyle: [],
  brandElements: [],
  mandatoryElements: [],
  forbiddenElements: [],
  cardCount: 1,
  cards: [
    { title: "", message: "", emphasis: "" }
  ],
  customSize: { width: "", height: "" },
  promptSettings: {
    addPreamble: true,
    koreanContent: true,
    outputMode: "block"
  }
}
```

저장 스키마에는 모드를 명시한다.

```js
{
  schemaVersion: 1,
  mode: "promotion",
  promotionState: { ... }
}
```

기존 발표자료 저장 데이터는 다음과 같이 유지한다.

```js
{
  schemaVersion: 3,
  mode: "presentation",
  ...
}
```

## 8. 단계별 구현 계획

## 8.1 1단계: 탭 구조 추가

목표:

- `홍보이미지` 최상위 탭 추가
- 빈 상태의 신규 패널 렌더
- 기존 발표자료/생성기 동작에 영향 없게 유지

작업:

- `index.html`에 `tabBtnPromotion`, `panePromotion` 추가
- `src/tabs.js`를 3탭 구조로 확장
- 헤더 액션 표시 규칙 정의

완료 기준:

- 세 탭 전환 정상
- 기존 스모크 테스트 통과

## 8.2 2단계: 홍보이미지 MVP UI

목표:

- 포스터/카드뉴스/SNS 서브 탭과 입력 폼 추가

작업:

- `src/promotion.js`에서 초기 상태 정의
- `assetType` 전환 UI 구현
- 공통 필드와 유형별 필드 렌더링

완료 기준:

- 각 유형 전환 시 필드가 다르게 표시됨

## 8.3 3단계: 프롬프트 생성 로직

목표:

- 홍보이미지 전용 프롬프트 생성기 구현

작업:

- 공통 프롬프트 템플릿 정의
- 포스터/카드뉴스/SNS별 조합 규칙 구현
- 우측 프리뷰에 실시간 반영

완료 기준:

- 유형별 프롬프트가 의미 있게 달라짐

## 8.4 4단계: 프리셋

목표:

- 빠르게 시작할 수 있는 프리셋 제공

초기 프리셋 후보:

- 행사 포스터
- 모집 공고 포스터
- 카드뉴스 시리즈 안내형
- 인스타 피드 프로모션
- 스토리 CTA형

완료 기준:

- 프리셋 클릭 시 입력값과 프롬프트가 함께 반영됨

## 8.5 5단계: 저장/불러오기

목표:

- 홍보이미지 작업 상태 저장/복원

작업:

- 모드 구분 포함 저장 구조 설계
- 발표자료와 충돌 없는 마이그레이션 처리

완료 기준:

- `promotion` 모드 round-trip 성공

## 8.6 6단계: 진단 규칙

목표:

- 잘못된 입력 조합에 대한 경고 제공

예시:

- 헤드라인 미입력
- CTA 없음
- 카드뉴스 컷 수와 실제 입력 수 불일치
- 플랫폼 미선택
- 포스터에 텍스트 과밀

완료 기준:

- 최소한의 warning/summary 표시

## 8.7 7단계: 자동 검증

목표:

- 홍보이미지 MVP 흐름을 스모크 테스트에 편입

테스트 범위:

- 탭 진입
- 유형 전환
- 프리셋 적용
- 프롬프트 생성
- 저장/불러오기

완료 기준:

- `npm run smoke:test`에서 promotion 흐름 포함 통과

## 10. 리스크와 대응

### 10.1 리스크

- 발표자료 로직과 홍보이미지 로직이 다시 섞일 수 있음
- 카드뉴스가 사실상 멀티 슬라이드 편집기로 비대해질 수 있음
- 저장 스키마가 빠르게 복잡해질 수 있음

### 10.2 대응

- `promotion.js`로 모듈 경계 강제
- MVP에서는 카드뉴스를 `시리즈 입력 + 전체 프롬프트 생성`까지만 제한
- 저장 데이터에 `mode` 필드 명시

---

## 11. 완료 기준

- 최상위 `홍보이미지` 탭이 추가된다.
- 포스터/카드뉴스/SNS 서브 유형이 동작한다.
- 유형별 입력 폼이 분리되어 보인다.
- 유형별 프롬프트가 생성된다.
- 프리셋이 최소 3개 이상 동작한다.
- 저장/불러오기에서 상태가 복원된다.
- 스모크 테스트가 통과한다.

---

## 12. 권장 개발 순서

1. 탭 구조 추가
2. 홍보이미지 빈 패널 + 상태 초기화
3. 포스터 MVP 완성
4. SNS 홍보이미지 추가
5. 카드뉴스 추가
6. 저장/불러오기 확장
7. 진단과 자동 테스트 추가

이 순서로 가면 가장 빨리 사용자 가치가 나오고, 중간에 구조가 흔들릴 가능성도 낮다.
