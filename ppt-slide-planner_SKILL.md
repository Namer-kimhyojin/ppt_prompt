# PPT 슬라이드 기획안 생성 스킬

원문 문서를 분석하여 슬라이드 1장씩 완성된 기획안을 MD 파일로 작성한다.

**이 스킬의 두 가지 핵심 목표:**
1. **비주얼** — 시청자가 화면을 보는 순간 메시지를 이해하는 슬라이드 구성
2. **스크립트** — 공식적이고 신뢰감 있으면서도, 읽는 즉시 자연스럽게 말할 수 있는 문장

> **역할 분담 원칙**
> - **이 스킬**: 레이아웃 구조·정보 배치·시선 흐름·강조 위계를 결정한다
> - **프롬프트덱**: 색상 팔레트·배경 명도·강조색 등 비주얼 스타일을 결정한다
> - 따라서 이 스킬의 비주얼 기획에는 특정 색상값·색상명을 지정하지 않는다.
>   강조 표현은 "시선 시작점", "강하게", "대비되게" 등 위계와 무게감 언어만 사용한다.

## 1. 작업 시작 전 확인

컨텍스트에 이미 있는 정보는 재질문하지 않는다. 없는 것만 확인한다.

| 항목 | 기본값 |
|------|--------|
| 소스 문서 | 첨부 파일 또는 입력 텍스트 |
| 총 슬라이드 수 | 문서 분량 기반 자동 제안 |
| 청중·목적 | 정부 심사위원회 |
| 발표 시간 | 슬라이드당 1분 기준 |
| 콘텐츠 밀도 | **표준(Standard)** — 아래 섹션 참고 |
| 디자인 | 기관 템플릿 적용 (색상·폰트는 프롬프트덱 팔레트 따름) |

## 2. 발표 전략 수립

### 2-1. 청중 분석
슬라이드를 만들기 전 청중이 누구인지 먼저 정의한다. 청중이 바뀌면 강조 포인트와 스크립트 어조가 달라진다.

| 청중 유형 | 핵심 관심사 | 설득 포인트 |
|---------|-----------|-----------|
| 정부 심사위원 | 타당성·파급효과·실현가능성 | 수치 근거 + 리스크 인식 + 실행 의지 |
| 장관·고위직 | 핵심 결론·정책 임팩트 | 1장 요약 → 3가지 이유 → 요청 |
| 업계·전문가 | 기술적 차별성·시장 논리 | 데이터 정밀도 + 업계 맥락 |
| 일반 시민·언론 | 생활과의 연결·직접 이익 | 쉬운 말 + 구체적 숫자 |

### 2-2. 논리 흐름 선택
```
[설득형]  Why(왜 필요한가) → What(무엇을) → How(어떻게) → So What(결과)   ← 심사·승인
[보고형]  현황 → 성과 → 과제 → 계획                                         ← 업무보고
[제안형]  문제 → 기회 → 해법 → 요청                                          ← 정책 제안
```

### 2-3. 슬라이드 배분표
작업 전 전체 목록을 먼저 제시하고, 확인 없이 바로 작성을 시작한다.

## 3. 콘텐츠 밀도 설계

슬라이드마다 정보량과 요약도를 의도적으로 조절한다.
**발표 전략 수립 단계에서 각 슬라이드의 밀도 레벨을 먼저 배정한 뒤 기획에 들어간다.**

### 3-1. 밀도 레벨 정의

| 레벨 | 명칭 | 화면 정보량 | 스크립트 역할 | 적합한 슬라이드 |
|------|------|-----------|------------|--------------|
| **L1** | **임팩트** | 키워드 1~2개 + 수치 1개 | 침묵이 더 많다. 화면이 말한다. | 표지 / 파트 전환 / 클로징 / 위기 선언 |
| **L2** | **집중** | 핵심 메시지 1개 + 근거 2~3개 | 맥락과 의미를 덧붙인다 | 핵심 주장 / Why Now / 핵심 성과 |
| **L3** | **표준** | 주제 1개 + 보조 항목 3~5개 | 항목을 해설하며 흐름을 잇는다 | 전략 설명 / 사업 구조 / 추진 현황 |
| **L4** | **정보** | 데이터·표·항목이 6개 이상 | 핵심만 짚고 나머지는 자료로 남긴다 | 예산 명세 / 비교 분석 / 현황 통계 |

### 3-2. 밀도 설계 원칙

**레벨 배분 가이드**
```
L1 임팩트   ────  전체의 15~20%  (도입·전환·마무리)
L2 집중     ────  전체의 25~30%  (핵심 설득 슬라이드)
L3 표준     ────  전체의 40~50%  (본론 중심)
L4 정보     ────  전체의 10~15%  (데이터·부록성 슬라이드)
```

**레벨 전환 패턴** — 같은 레벨이 3장 이상 연속되면 청중이 지친다.
```
나쁜 예: L3 → L3 → L3 → L3 → L3
좋은 예: L3 → L3 → L2(정점) → L3 → L1(파트 전환)
```

**밀도와 스크립트의 관계**

| 레벨 | 발화 문장 수 | 침묵 활용 |
|------|-----------|---------|
| L1 | 1~2문장 | Hook 후 2~3초 |
| L2 | 3~4문장 | Hook 후 1~2초 |
| L3 | 4~5문장 | 필요 시 1초 |
| L4 | 3~4문장 (핵심만) | 없음 |

> **L4 주의:** 정보가 많아도 스크립트는 짧게. 화면이 데이터를 담당하고, 발표자는 "어떻게 읽어야 하는가"만 말한다.

### 3-3. 슬라이드 배분표에 밀도 레벨 포함

배분표 출력 시 레벨 컬럼을 반드시 포함한다.

```markdown
| No | 제목 | 파트 | 논리 위치 | 레이아웃 유형 | 밀도 |
|----|------|------|---------|------------|------|
| 01 | 표지 | 도입 | — | 풀임팩트 | L1 |
| 02 | 핵심 메시지 | 도입 | Why | 빅타이포 임팩트 | L2 |
...
```

**컨셉:**

### 🎤 발표 스크립트

스크립트는 **큐 시트 형식**으로 작성한다.
발화(말할 내용)와 연출 큐(시각·타이밍 지시)를 한 줄씩 교차 배치한다.

연출 큐 기호:
- `[슬라이드 전환]` — 이 시점에 다음 슬라이드로 넘긴다
- `[화면 → {요소}]` — 해당 요소를 레이저포인터·클릭으로 지시한다
- `[침묵 N초]` — 말을 멈추고 청중이 받아들이는 시간을 준다
- `[청중 시선]` — 화면에서 눈을 떼고 청중 전체를 바라본다
- `[속도: 느리게/빠르게]` — 해당 문장의 발화 속도를 조절한다

```
🪝 Hook
[청중 시선]
"{수치 / 대비 / 선언 — 한 문장으로. 화면이 아닌 청중을 향해.}"
[침묵 1~2초]

📖 Body  ← 2~3개 덩어리. 각 덩어리는 한 문장 안에 맥락을 통째로 담는다.
[화면 → {시선 시작점 요소}]
"{덩어리 ①: 현실 또는 배경. ~지만 / ~가운데 / ~이지만 으로 대비나 맥락을 한 문장에.}"

[화면 → {두 번째 강조 요소}]
"{덩어리 ②: 위기 또는 의미. ~는 순간 / ~되면 으로 조건-결과를 한 문장에.}"
[침묵 1초]

"{덩어리 ③: 결론 또는 필요. 짧고 단호하게.}"

🌉 Bridge  ← ①이 슬라이드 결론 닫기 → ②다음 슬라이드 열기
[청중 시선]
[속도: 느리게]
"{①닫기: 지금 슬라이드의 핵심을 한 문장으로 맺는다.}"
"{②열기: 다음 슬라이드로 자연스럽게 이어지는 한 문장.}"
[슬라이드 전환]
```

## 5. 비주얼 기획 가이드

> **색상 분리 원칙:** 이 가이드에서 레이아웃 유형명과 구조 설명은 형태·구성·위계만 기술한다.
> 배경 명도(밝음/어두움), 팔레트, 강조색 지정은 프롬프트덱의 역할이다.
> "고대비 배경", "저채도 배경" 등의 표현은 방향 힌트이며, 실제 색상값은 프롬프트덱에서 결정한다.

### 5-1. 레이아웃 유형 라이브러리

슬라이드마다 다른 레이아웃을 사용한다. **같은 유형을 3장 연속 쓰지 않는다.**

#### 임팩트·선언형 (L1·L2에 적합)

| 유형 | 구조 | 적합한 슬라이드 |
|------|------|--------------|
| **풀임팩트** | 전면 고대비 배경 + 대형 타이포 + 넓은 여백 / 배경은 콘텐츠 대비 최고 무게감이어야 함 — 명도·색상은 팔레트 따름 | 표지·파트 전환·클로징 |
| **빅타이포 임팩트** | 핵심 수치 1개가 화면 40~50% 차지 + 보조 Bento 카드 | Why Now·충격 수치·위기 선언 |
| **풀블리드 인용** | 배경 이미지 전면 + 반투명 오버레이 + 한 줄 인용문 | 비전 선언·도입 충격·클로징 |
| **카운트다운 카드** | 숫자 3~4개를 대형으로 나란히 배치 + 각각 짧은 레이블 | 3대 이유·4대 전략 선언 |

#### 대비·비교형 (L2·L3에 적합)

| 유형 | 구조 | 적합한 슬라이드 |
|------|------|--------------|
| **대비 분할** | 좌우를 두 영역으로 나눔 (시각 무게감 대비로 문제↔해법 구분, 명도 차이는 팔레트 따름) | 현황 vs 목표·문제 vs 해법 |
| **Before·After 스플릿** | 상하 또는 좌우로 시간 전후를 명확히 분리 | 정책 효과·사업 전후 변화 |
| **경쟁 매트릭스** | 2×2 또는 3×3 그리드로 항목·특성 비교 | 경쟁사 비교·입지 선정·기술 비교 |
| **선택지 카드** | 동등한 옵션 2~3개를 카드로 병렬 배치, 추천 옵션 시각 위계 강조 | 전략 선택·시나리오 제시 |

#### 구조·흐름형 (L3에 적합)

| 유형 | 구조 | 적합한 슬라이드 |
|------|------|--------------|
| **서사 흐름** | 수평 5단계 플로우 (화살표·아이콘·키워드) | 프로세스·밸류체인·추진 단계 |
| **허브-스포크** | 중심 노드 + 연결 요소 4~6개 방사형 배치 | 거버넌스·협력체계·생태계 |
| **타임라인 드라마** | 수평 시간 축 + 노드별 핵심 카드 | 로드맵·추진일정·단계별 목표 |

| **피라미드·계층** | 상위→하위 계층 구조를 피라미드 또는 트리로 표현 | 조직도·정책 체계·목표-수단 연결 |
| **순환 사이클** | 원형 또는 타원형 순환 고리 + 단계 레이블 | 생태계 선순환·반복 프로세스 |
| **스윔레인** | 행별 주체 + 열별 단계로 역할·책임 분리 | 협업 프로세스·부처 간 역할 분담 |

#### 공간·지역형 (L3에 적합)

| 유형 | 구조 | 적합한 슬라이드 |
|------|------|--------------|
| **지도 + 카드** | 상단 지형·지도 + 하단 팩트 카드 그리드 | 입지·공간구조·권역 설명 |
| **클러스터 맵** | 지도 위에 거점·노드·연결선을 오버레이 | 산업 클러스터·물류망·협력 권역 |

#### 데이터·분석형 (L3·L4에 적합)

| 유형 | 구조 | 적합한 슬라이드 |
|------|------|--------------|
| **도넛 + Bento** | 좌: 예산/비중 도넛차트 + 우: 항목 카드 그리드 | 예산 배분·4대 사업 요약 |
| **데이터 해부** | 중앙 70% 표·차트 + 우측 30% 해석 코멘트 | 취약성 분석·현황표·성과 비교 |
| **트렌드 라인** | 꺾은선 또는 영역 차트 + 주요 이벤트 마커 | 시장 성장·정책 효과 추이 |
| **히트맵 그리드** | 행·열 교차점을 채움 강도로 표현 (강도 표현 방식은 팔레트 따름) | 지역별·품목별 현황·리스크 매트릭스 |
| **스코어카드** | KPI 항목별 현재값·목표값·달성률을 카드로 나열 | 성과 현황·중간 점검·목표 대비 |
| **모자이크 타일** | 크기가 서로 다른 타일로 비중·중요도를 면적으로 표현 | 시장 점유율·예산 비중·항목 구성 |

### 5-2. 강조 설계 원칙
- **시선 시작점은 슬라이드당 1개** — 두 개 이상이면 아무것도 강조되지 않는다
- 시선의 자연스러운 흐름: **좌상→우→하** (Z패턴) 또는 **중앙→주변** (원형)
- 여백은 디자인 요소다 — 빽빽한 슬라이드는 메시지를 죽인다
- 파트 전환 슬라이드는 반드시 **풀임팩트**형으로 리듬을 환기한다
- **밀도 레벨과 레이아웃을 함께 고려** — L1에 데이터 해부형은 어울리지 않는다
- **강조는 크기·굵기·여백으로 표현한다** — 색상 지정은 프롬프트덱의 역할이다

## 6. 발표 스크립트 작성 원칙

### 6-0. 이론적 기반 — 세 가지 프레임워크

이 스킬의 스크립트 원칙은 다음 세 이론을 통합한 것이다. 각각이 해결하는 문제가 다르다.

| 프레임워크 | 출처 | 이 스킬에서 담당하는 역할 |
|----------|------|----------------------|
| **SCR** (Situation–Complication–Resolution) | McKinsey & Company | **슬라이드 내 스크립트 흐름** — 현실→문제→해법 순서로 청중을 논리적으로 이끈다 |
| **Pyramid Principle** (결론 먼저) | Barbara Minto, McKinsey | **Hook 설계** — 핵심 메시지를 먼저 던지고 근거를 뒤에 배치한다 |
| **Oral Style** (구어체 원칙) | 언어학 연구 (Tannen 외) | **문장 단위** — 발표 스크립트는 읽히는 글이 아니라 들리는 말이다 |

#### SCR — 슬라이드 스크립트의 뼈대

SCR은 Situation(현재 상황)·Complication(그 상황이 행동을 요구하는 이유)·Resolution(해법 또는 요청)의 세 부분으로 메시지를 구조화하는 방법이다. 슬라이드 1장의 스크립트도 이 흐름을 따른다.

```
Situation  →  "배터리는 국내에서 만들지만, 핵심 소재의 대부분은 중국에서 들어오고 있습니다."
Complication → "이 공급망이 막히는 순간, 이차전지 생태계 전체가 멈춰버립니다."
Resolution  →  "지금 우리에게는 소재를 직접 만들 수 있는 기반이 필요합니다."
```

Hook은 Situation의 가장 날카로운 한 줄, Body는 S→C 흐름, Bridge의 ①닫기는 Resolution이다.

#### Pyramid Principle — 결론을 먼저

Minto는 "생각은 아래에서 위로 하지만, 발표는 위에서 아래로 한다"고 말했다. McKinsey에서 "결론부터"는 모든 신입에게 주입되는 원칙이다. 발표 스크립트에서 이것은 두 가지를 의미한다.

- **Hook = 핵심 메시지 먼저.** 수치나 결론을 첫 문장에 배치한다. "95%."처럼.
- **Body = 근거는 뒤에.** Hook이 던진 주장을 S→C 순서로 뒷받침한다.

#### Oral Style — 말은 글과 다르다

구어체는 문어체에 비해 1·2인칭 대명사를 더 많이 쓰고, 사고 단위(thought unit)가 짧으며, 단음절 단어와 친숙한 어휘를 더 많이 사용한다. 발표 스크립트에서 이것이 뜻하는 바는 다음과 같다.

- **"저희", "우리", "포항은"** — 피동형·공문서 주어 대신 구체적인 주어
- **덩어리 단위** — 문장을 잘게 끊는 대신, 하나의 생각을 한 문장 안에 통째로 담기
- **핵심 표현은 반복해도 된다** — 청중은 되돌아가 읽을 수 없다. 같은 키워드를 2~3번 반복하는 것은 결함이 아니라 전략이다

### 6-1. 어조 목표: "공식적이되 자연스러운 구어체"

스크립트의 목표는 **읽는 즉시 말할 수 있는 문장**이다.
격식은 유지하되, 듣는 사람이 논리를 따라올 수 있어야 한다.

| 특성 | 나쁜 예 | 좋은 예 |
|------|--------|--------|
| **구체적 주어** | "추진되어 왔으며 향후에도 지속될 예정입니다." | "포항시가 추진해 왔고, 앞으로도 이어갑니다." |
| **덩어리 문장** | "만듭니다. / 옵니다. / 멈춥니다." (3개 선언) | "만들지만, 소재는 중국에서 들어오고 있습니다." (1개 덩어리) |
| **결론 먼저** | "시장 분석 결과, LFP가 성장하고 있어 대응이 필요합니다." | "LFP 시장은 3년 만에 두 배로 커졌습니다. 지금이 진입 시점입니다." |
| **쉬운 어휘** | "구조적 취약성" | "언제든 흔들릴 수 있는 구조" |
| **전문용어 처리** | "LMFP가 주목받고 있습니다." | "LMFP, 즉 리튬망간철인산염이 주목받고 있습니다." |

### 6-2. 쓰지 않는 표현 / 대체 표현

| ❌ 피할 표현 | 이유 | ✅ 대신 이렇게 |
|------------|------|--------------|
| "말씀드리자면", "사실은" | 시작을 늦추는 군더더기 | 바로 본론으로 |
| "~이라고 할 수 있겠습니다" | 확신이 없어 보인다 | "~입니다", "~를 보여줍니다" |
| "그리고", "또한", "아울러" | 단순 나열 접속사 | 덩어리로 묶거나 마침표로 끊기 |
| "~에 대해서 말씀드리겠습니다" | Hook 없이 예고만 | Hook으로 바로 시작 |
| "보시다시피 차트에서..." | 화면을 낭독하는 것 | 수치의 의미를 말한다 |
| "본 단지는", "해당 사업은" | 딱딱한 공문서체 | "저희는", "포항은", "이 사업은" |
| "이상으로 발표를 마치겠습니다" | 힘 빠지는 마무리 | 행동 요청으로 끝낸다 |
| "~라고 생각합니다" | 확신이 없어 보인다 | "~입니다", "~라고 봅니다" |
| "추진되어 왔으며 향후에도" | 피동형 중첩 | 능동형으로 바꾼다 |

#### 수사적 선언 — 사실이 말하게 둔다

발표자가 직접 "중요하다", "시급하다", "유일하다"고 선언하는 문장은 광고 문구처럼 들린다. 사실을 담담하게 말하면 청중이 스스로 판단한다. **발표자의 역할은 중요성을 선언하는 것이 아니라, 사실을 정확하게 전달하는 것이다.**

| ❌ 수사적 선언 (광고 문구처럼 들림) | ✅ 사실로 대체 (청중이 스스로 판단) |
|--------------------------------|-------------------------------|
| "지금 그것이 가장 시급합니다." | "지금 국내에 LFP 소재를 양산할 수 있는 거점이 없습니다." |
| "소재 자립의 시작 — 포항입니다." | "포항입니다." |
| "95%의 의존을 끊는 일은, 오늘 이 결정에서 시작됩니다." | "입지가 있고, 기업이 있고, 계획이 있습니다." |
| "포항이 유일한 답입니다." | "이 세 조건이 한 도시에 모인 곳은 포항뿐입니다." |
| "역사적인 결정을 내려 주십시오." | "지정해 주십시오." |

> **원칙:** 슬로건·과장·감정 호소는 모두 수사다. 사실의 밀도가 높을수록 담담해지고, 담담할수록 오히려 무게가 생긴다. 클로징일수록 더 짧고 더 담담하게.

### 6-3. Hook → Body(SCR) → Bridge 구조

| 단계 | SCR 매핑 | 역할 | 분량 |
|------|---------|------|------|
| **🪝 Hook** | Situation의 핵심 한 줄 | 결론·수치·대비를 먼저 던져 청중을 집중시킨다 | 1문장 |
| **📖 Body** | S → C 흐름 | 현실을 깔고(S), 왜 문제인지를 드러낸다(C) | **2~3개 덩어리** |
| **🌉 Bridge** | Resolution으로 닫기 + 다음 열기 | ①해법·결론으로 이 슬라이드를 맺고 ②다음으로 넘긴다 | 1~2문장 |

#### Body — SCR 덩어리로 쓰기

각 덩어리는 문장을 잘게 끊는 대신, 맥락을 한 문장 안에 통째로 담는다.

```
❌ 선언 나열 (SCR 없이 끊기는 느낌)
"배터리는 국내에서 만듭니다."           ← 사실만
"핵심 소재는 중국에서 옵니다."          ← 사실만
"공급이 막히면 생산이 멈춥니다."        ← 사실만

✅ SCR 덩어리 (논리가 흐르는 느낌)
"배터리는 국내에서 만들지만,
 핵심 소재의 대부분은 중국에서 들어오고 있습니다."     ← S: 현실 (대비를 한 문장에)

"이 공급망이 막히는 순간,
 이차전지 생태계 전체가 멈춰버립니다."                 ← C: 문제 (조건-결과를 한 문장에)

"지금 우리에게는 소재를 직접 만들 수 있는 기반이 필요합니다."  ← R: 해법 (Bridge로 넘어가는 연결)
```

**한 문장 안에 맥락을 담는 방법**

| 관계 | 문장 내 연결 방식 | 예시 |
|------|--------------|------|
| 대비 S (A이지만 B) | `~지만, ~` / `~하지만 ~` | "생산 능력은 갖췄지만, 원료 자립도는 아직 5%에 불과합니다." |
| 조건-결과 C (A이면 B) | `~는 순간, ~` / `~되면, ~` | "공급망이 흔들리는 순간, 완성차 라인 전체가 멈춥니다." |
| 배경-핵심 S (A 가운데 B) | `~가운데, ~` / `~상황에서, ~` | "시장이 빠르게 커지는 가운데, 소재 의존도는 여전히 95%입니다." |
| 나열-결론 R | `세 가지 모두` / `~으로` | "입지·인프라·인력, 세 가지 조건이 포항에 모여 있습니다." |

> **핵심 키워드는 반복해도 된다.** 청중은 되돌아가 읽을 수 없다. "소재 자립"처럼 이 발표의 핵심어는 슬라이드마다 반복 등장시켜 각인시킨다.

#### Bridge — ①Resolution 닫기 + ②다음 열기

```
❌ 예고만 하는 Bridge (Resolution 없이 끊기는 느낌)
"그렇다면 포항이 왜 이 문제의 답이 될 수 있는지 보겠습니다."

✅ Resolution 닫기 + 열기
"지금 우리에게 필요한 건,
 소재를 직접 만들 수 있는 기반입니다."        ← ①R 닫기: 해법으로 이 슬라이드를 맺는다
"포항이 바로 그 기반을 갖추고 있습니다."      ← ②열기: 다음 슬라이드로 자연스럽게 이어진다
[슬라이드 전환]
```

**Bridge ①닫기 패턴**

| 유형 | 예시 |
|------|------|
| 해법 선언 | "지금 우리에게 필요한 건, 원료를 직접 만들 수 있는 기반입니다." |
| 결론 한 줄 | "결국 이 문제의 핵심은 소재 자립, 그 하나입니다." |
| 문제 규정 | "소재를 직접 만들 수 있느냐, 없느냐 — 그것이 경쟁의 갈림길입니다." |

### 6-4. Hook 유형별 예시 (결론 먼저 — Pyramid Principle)

피라미드 원칙에서 결론부터 말한다는 것은, 청중에게 먼저 답을 주고 그 근거를 뒤에 배치하는 것이다. Hook은 이 원칙의 첫 번째 실현이다.

| 유형 | 예시 문장 |
|------|---------|
| 충격 수치형 | "95%. 핵심 배터리 원료의 중국 의존도입니다." |
| 결론 선언형 | "소재 자립 없이는 배터리 강국도 없습니다." |
| 대비형 | "배터리는 만들 수 있습니다. 원료는 만들지 못합니다." |
| 질문형 | "그 원료를, 지금 어디서 가져오고 있을까요?" |
| 공감형 | "공급망 위기는 뉴스 속 이야기가 아닙니다. 지금 우리가 마주한 현실입니다." |

### 6-5. Bridge 유형별 예시 (2단 구조)

| 상황 | ①R 닫기 (해법·결론) | ②열기 (다음 슬라이드 예고) |
|------|------------------|------------------------|
| 문제→해법 전환 | "소재 자립. 그것이 지금 포항이 해야 할 일입니다." | "포항은 그 조건을 이미 갖추고 있습니다." |
| 전략→실행 전환 | "전략의 방향은 명확합니다." | "이제 그것을 실현할 구체적인 계획을 보여드리겠습니다." |
| 근거→결론 전환 | "입지·기업·인프라, 세 조건이 모두 포항에 있습니다." | "남은 질문은 하나입니다 — 언제 시작하느냐." |
| 파트 전환 | "이상이 저희가 진단한 현황입니다." | "다음은 그에 대한 포항의 대응 전략입니다." |

### 6-6. 시각 연출 큐 원칙

| 큐 기호 | 의미 | 사용 시점 |
|--------|------|---------|
| `[슬라이드 전환]` | 다음 슬라이드로 넘긴다 | Bridge 발화 직후 |
| `[화면 → {요소명}]` | 해당 요소를 포인팅·클릭으로 지시 | 수치·도표를 언급하는 직전 또는 직후 |
| `[침묵 N초]` | 말을 멈추고 청중이 받아들이는 시간 | Hook 직후, 핵심 수치 직후 |
| `[청중 시선]` | 화면에서 눈을 떼고 청중 전체를 바라본다 | Hook 시작, Bridge 마지막 |
| `[속도: 느리게]` | 해당 문장을 천천히 말한다 | 클로징, 핵심 선언 |
| `[속도: 빠르게]` | 해당 문장을 빠르게 말한다 | 배경 설명, 보조 데이터 나열 |

**3원칙**
1. **포인팅은 말과 동시에** — "20억 원을 투입합니다"라고 말하는 순간 `[화면 → 해당 항목]`
2. **침묵은 연출이다** — Hook 직후 2초 침묵은 청중의 주의를 끌어모은다
3. **클로징은 화면 없이** — 마지막 문장은 `[청중 시선]`으로 화면이 아닌 사람을 본다

### 6-7. 발표 리듬 설계

```
[고조]   위기·긴급성 슬라이드  — 짧은 문장, 빠른 호흡, 수치 직격
[안정]   해법·구조 슬라이드    — 차분하고 명확한 설명, 논리 전개
[정점]   핵심 임팩트 슬라이드  — 슬라이드 전환 후 1~2초 침묵, 그 다음 한 문장
[마무리] 클로징 슬라이드      — 가장 느린 속도, 청중 전체를 바라보며 마지막 요청
```

> - 구성: {N}장 / 발표 시간 약 {N}분
> - 디자인: 기관 템플릿 적용 (색상·팔레트는 프롬프트덱에서 별도 설정)

## 8. 품질 체크리스트

출력 전 각 슬라이드에 대해 확인한다.

**밀도**
- [ ] 슬라이드 목록에 밀도 레벨(L1~L4)이 배정됐는가
- [ ] L1·L2가 전체의 40~50% 이내인가
- [ ] 같은 레벨이 3장 이상 연속되지 않는가
- [ ] L4 슬라이드의 스크립트가 핵심만 짚고 있는가

**비주얼**
- [ ] 시선 시작점(★)이 슬라이드당 1개로 명확한가
- [ ] AI 명세에 위치·높이·콘텐츠 유형·내용이 모두 있는가
- [ ] 같은 레이아웃 유형이 3장 연속으로 쓰이지 않는가
- [ ] 파트 전환 슬라이드가 풀임팩트형인가
- [ ] 밀도 레벨에 맞는 레이아웃을 선택했는가
- [ ] AI 명세에 특정 색상값·색상명이 포함되지 않았는가 (색상은 프롬프트덱 역할)
- [ ] 강조 표현이 크기·굵기·여백 언어로만 기술됐는가

**스크립트**
- [ ] Hook이 화면이 아닌 청중을 향해 던지는 형태인가
- [ ] Body가 2~3개 덩어리로 구성됐는가 (잘게 끊긴 문장 나열이 아닌가)
- [ ] 각 덩어리가 맥락을 한 문장 안에 통째로 담고 있는가 (~지만 / ~는 순간 / ~가운데 등)
- [ ] Body가 화면 내용을 낭독하지 않고 의미를 해석하는가
- [ ] 발화 덩어리 수가 밀도 레벨 기준에 맞는가
- [ ] 군더더기 표현이 없는가 (6-2 목록 참고)
- [ ] 수사적 선언이 없는가 — "가장 시급합니다", "유일한 답", "역사적" 등 발표자가 중요성을 직접 선언하는 문장 (6-2 참고)
- [ ] 피동형·공문서체 표현이 능동형·구어체로 바뀌었는가
- [ ] Bridge가 ①결론 닫기 + ②다음 열기의 2단 구조인가
- [ ] `[화면 → {요소}]` 큐가 수치·도표 언급과 연동됐는가
- [ ] `[침묵]` 큐가 Hook 직후 또는 핵심 수치 직후에 있는가
- [ ] 클로징 마지막 문장에 `[청중 시선]` 큐가 있는가
- [ ] 클로징이 요약이 아닌 행동 요청으로 끝나는가
- [ ] 클로징이 짧고 담담한가 — 마무리일수록 문장 수를 줄인다

**흐름**
- [ ] Bridge가 다음 슬라이드를 자연스럽게 예고하는가
- [ ] 발표 전체가 고조→안정→정점→마무리 리듬을 따르는가
- [ ] MD 파일로 출력됐는가
